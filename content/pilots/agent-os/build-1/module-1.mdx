# Module 1: Install vault-intake

vault-intake runs as a Claude skill. There are three ways to use it depending on your setup. Pick the one that matches how you already work with Claude.

## Choose your path

**CLI path** -- for developers comfortable in a terminal. vault-intake runs directly as a CLI tool invoked from your shell or from a script. Most control, most flexibility.

**Claude Desktop path** -- for users who run Claude Desktop with MCP servers. vault-intake is added as an MCP server in your Claude Desktop config. You invoke it through Claude's chat interface.

**Cowork path** -- for users inside a Cowork environment or similar AI-assisted workspace where Claude Code is available. vault-intake runs as a skill within your existing Claude Code session.

You only need one path. Start with the one you already have set up.

## CLI installation

<Action>
**Prerequisites:** Node.js 18+ installed; an Anthropic API key set as `ANTHROPIC_API_KEY` in your environment.
</Action>

```bash
npm install -g vault-intake
```

Verify the install:

```bash
vault-intake --version
```

You should see a version number. If you see a "command not found" error, check that your npm global bin directory is on your PATH.

<WatchOut>
On Windows, the npm global directory is usually `%APPDATA%\npm`. On Mac/Linux it's typically `/usr/local/bin` or `~/.npm-global/bin`. If `vault-intake --version` fails, run `npm config get prefix` to find where global packages are installed and add `/bin` to your PATH.
</WatchOut>

## Claude Desktop (MCP) installation

<Action>
**Prerequisites:** Claude Desktop installed; the MCP configuration file accessible at `~/Library/Application Support/Claude/claude_desktop_config.json` (Mac) or `%APPDATA%\Claude\claude_desktop_config.json` (Windows).
</Action>

Add vault-intake as an MCP server in your Claude Desktop config:

```json
{
  "mcpServers": {
    "vault-intake": {
      "command": "npx",
      "args": ["-y", "vault-intake"],
      "env": {
        "ANTHROPIC_API_KEY": "your-key-here"
      }
    }
  }
}
```

Restart Claude Desktop after saving. You should see vault-intake appear in the available tools list.

<WatchOut>
Do not put your API key directly in the config file if you share your machine or sync your config. Use an environment variable reference or a secrets manager instead.
</WatchOut>

## Cowork / Claude Code installation

If you're using Claude Code (the CLI) or a Cowork environment:

```bash
claude skills install vault-intake
```

Or if skills are managed via your session, vault-intake is available as a skill named `vault-intake` once installed. Invoke it by name in your Claude Code session.

## Verify your installation

Regardless of path, run a quick verification:

**CLI:**
```bash
vault-intake --version
vault-intake test
```

**Claude Desktop:** Open a new chat and type "test vault-intake" -- you should see the skill respond with its version and vault path.

**Cowork:** In your Claude Code session, invoke the skill and ask it to report its version.

<KeyData>
A successful verification shows: vault-intake version number, the resolved vault path (from your CLAUDE.md Vault Config block, or a default), and confirmation that the Anthropic API key is valid.
</KeyData>

If you see a vault path you don't recognize, that's expected if you haven't written your Vault Config block yet -- Module 2 covers that. The default path is `~/vault` or `~/Documents/Obsidian`.
