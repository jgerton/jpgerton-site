# Build 1: First Capture

This build covers one thing: getting vault-intake running on your machine and taking a real piece of content through the pipeline.

Five modules, one exercise each. Total time: 1-2 hours depending on your setup path.

## What this build covers

1. **Install vault-intake** -- get it running on your machine using CLI, Claude Desktop, or Cowork
2. **Configure your vault** -- write the CLAUDE.md Vault Config block that tells vault-intake about your domains and preferences
3. **Your first capture** -- take a real piece of content through the pipeline from source to note
4. **Friction reporting** -- structured feedback on what felt slow, confusing, or broken
5. **Wishes, edge cases, and surprises** -- forward-looking signal for Build 2

## What this build does NOT cover

Advanced workflows, multi-source pipelines, and automation come later. Build 1 is about the baseline: does it work for your setup, does it land your content correctly, and what did you learn from the first real use.

## How exercises work

Each module ends with one exercise. Submit your response using the web form (auto-saves as you type) or via email. There are no right answers, only useful ones. Short, honest responses are better than long, polished ones.

Your responses are read directly. Patterns that surface across multiple pilots inform Build 2 scope. If something you report changes a design decision, you'll hear about it.

## How to use the module feedback

At the bottom of each module there's a short feedback form: readiness check plus two open fields. Fill it out before moving to the next module. It helps calibrate whether the module content is doing its job.

Start with [Module 1: Install vault-intake](/pilots/docs/agent-os/build-1/module-1).
