# Pilots Hub Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/pilots` hub to jpgerton.com where users sign in via magic link to access gated project content (first project: Freemium Playbook Build 1).

**Architecture:** Convex Auth with Resend magic links handles authentication. Convex stores pilot signups and project metadata. Next.js middleware protects `/pilots/[project]/[build]` routes. Playbook content lives as MDX files in the repo, rendered with the site's existing design tokens. The `/pilots` route is reusable for future projects beyond the playbook.

**Tech Stack:** Next.js 16 (app router), Convex (database + auth), Convex Auth (magic links via Resend), Tailwind 4, MDX for content rendering.

**CLI tools available:** `bunx convex` (v1.35.1) for Convex operations (init, dev, deploy, env set, data seeding). `vercel` (v48.10.3, global) for deployment and env vars. `bun` for package management. Use these CLIs directly instead of dashboard UIs where possible.

**Current codebase:** jpgerton-site at `E:\Projects\jpgerton-site`. Next.js 16, React 19, Tailwind 4. No Convex or auth currently installed. App router with routes: `/`, `/about`, `/contact`. Design tokens in `globals.css` (fluid typography, color system, spacing scale). Fonts: Space Grotesk (headings), Inter (body), JetBrains Mono (code).

**IMPORTANT:** This project uses Next.js 16 which has breaking changes from earlier versions. Before writing any code, read the relevant guide in `node_modules/next/dist/docs/` to verify API compatibility. Use `bun` for all package operations (never npm/npx).

```typescript
  pilotProjects: defineTable({
    slug: v.string(),
    title: v.string(),

    status: v.union(v.literal("active"), v.literal("coming-soon"), v.literal("archived")),

  pilotBuilds: defineTable({
    projectSlug: v.string(),
    buildSlug: v.string(),
    title: v.string(),
    contentPath: v.string(),
    order: v.number(),
    status: v.union(v.literal("draft"), v.literal("published")),
  })
    .index("by_project", ["projectSlug"])
    .index("by_slugs", ["projectSlug", "buildSlug"]),
});
```

```typescript
import { query } from "./_generated/server";
import { v } from "convex/values";

export const listActiveProjects = query({
  args: {},
  handler: async (ctx) => {
    return await ctx.db
      .query("pilotProjects")
      .filter((q) => q.eq(q.field("status"), "active"))
      .collect();
  },
});

export const getProject = query({
  args: { slug: v.string() },
  handler: async (ctx, args) => {
    return await ctx.db
      .query("pilotProjects")
      .withIndex("by_slug", (q) => q.eq("slug", args.slug))
      .first();
  },
});

export const getBuildsForProject = query({
  args: { projectSlug: v.string() },
  handler: async (ctx, args) => {
    return await ctx.db
      .query("pilotBuilds")
      .withIndex("by_project", (q) => q.eq("projectSlug", args.projectSlug))
      .filter((q) => q.eq(q.field("status"), "published"))
      .collect();
  },
});

export const getBuild = query({
  args: { projectSlug: v.string(), buildSlug: v.string() },
  handler: async (ctx, args) => {
    return await ctx.db
      .query("pilotBuilds")
      .withIndex("by_slugs", (q) =>
        q.eq("projectSlug", args.projectSlug).eq("buildSlug", args.buildSlug)

      .first();
  },
});
```

- [ ] **Step 3: Create middleware for route protection**

Create `middleware.ts` at project root:

```typescript
import {
  convexAuthNextjsMiddleware,
  createRouteMatcher,
  nextjsMiddlewareRedirect,
} from "@convex-dev/auth/nextjs/server";

const isSignInPage = createRouteMatcher(["/pilots/signin"]);
const isProtectedRoute = createRouteMatcher(["/pilots/*/*"]);

export default convexAuthNextjsMiddleware(async (request, { convexAuth }) => {
  if (isSignInPage(request) && (await convexAuth.isAuthenticated())) {
    return nextjsMiddlewareRedirect(request, "/pilots");
  }
  if (isProtectedRoute(request) && !(await convexAuth.isAuthenticated())) {
    return nextjsMiddlewareRedirect(request, "/pilots/signin");
  }
});

export const config = {
  matcher: ["/((?!.*\\..*|_next).*)", "/", "/(api|trpc)(.*)"],
};
```

This protects `/pilots/[project]/[build]` routes (two levels deep) while keeping `/pilots` and `/pilots/signin` public.

- [ ] **Step 1: Create magic link sign-in form**

Create `components/sign-in-form.tsx`:

```typescript
"use client";

import { useAuthActions } from "@convex-dev/auth/react";
import { useState } from "react";

export function SignInForm() {
  const { signIn } = useAuthActions();
  const [email, setEmail] = useState("");
  const [sent, setSent] = useState(false);
  const [error, setError] = useState("");

  async function handleSubmit(event: React.FormEvent) {
    event.preventDefault();
    setError("");
    try {
      const formData = new FormData();
      formData.set("email", email);
      await signIn("resend", formData);
      setSent(true);
    } catch {
      setError("Something went wrong. Try again.");
    }
  }

  if (sent) {
    return (
      <div className="text-center space-y-4">
        <h2 className="text-2xl font-heading font-bold">Check your email</h2>
        <p className="text-muted">
          We sent a sign-in link to <strong>{email}</strong>. Click it to access
          the pilots hub.
        </p>
        <p className="text-sm text-muted">
          Don&apos;t see it? Check your spam folder.
        </p>
      </div>
    );
  }

  return (
    <form onSubmit={handleSubmit} className="space-y-4 max-w-md mx-auto">
      <div>
        <label htmlFor="email" className="block text-sm font-medium mb-1">
          Email address
        </label>
        <input
          id="email"
          name="email"
          type="email"
          required
          value={email}
          onChange={(e) => setEmail(e.target.value)}
          placeholder="you@example.com"
          className="w-full px-4 py-3 rounded-lg border border-border bg-surface text-foreground placeholder:text-muted focus:outline-none focus:ring-2 focus:ring-accent"
        />
      </div>
      {error && <p className="text-sm text-red-500">{error}</p>}
      <button
        type="submit"
        className="w-full px-4 py-3 rounded-lg bg-accent text-white font-medium hover:opacity-90 transition-opacity"
      >
        Send magic link
      </button>
    </form>
  );
}
```

## Notes

- **Resend account needed:** Create a free account at resend.com. Free tier: 3,000 emails/month, 100 emails/day. More than enough for pilots.
- **Resend domain verification:** For magic links to not land in spam, verify your domain (jpgerton.com) in Resend's dashboard. Without verification, emails send from onboarding@resend.dev which works for testing but may hit spam filters.
- **Convex Auth is beta:** If it breaks in a future update, the fallback is Clerk (has official Convex integration, 50K free users). The migration would touch `convex/auth.ts`, `components/convex-provider.tsx`, `middleware.ts`, and `components/sign-in-form.tsx`. Everything else stays the same.
- **Content updates:** To update playbook content, edit `content/pilots/freemium-playbook/build-1.mdx` and push. No database changes needed. To add Build 2, create `build-2.mdx` and insert a new row in `pilotBuilds`.
- **Future projects:** To add a new pilot project, insert a row in `pilotProjects` and create a new directory under `content/pilots/[slug]/`. The pages are dynamic and pick up new projects automatically.
