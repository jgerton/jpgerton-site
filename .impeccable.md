## Design Context

### Users
Claude Code practitioners and AI-curious developers who want to level up their tooling. They arrive from YouTube, X, or the YCAH Skool community. Context: they're mid-workflow, looking for something they can install or learn right now. They respect craft and can smell marketing filler instantly.

### Brand Personality
**Warm, technical, opinionated.** Approachable but has strong views backed by experience. Shares freely, demonstrates through action. The site should feel like visiting a competent craftsman's workshop: calm confidence, tools well-organized, projects visible, nothing performative.

### Aesthetic Direction
- **Feel:** Competent craftsman. No flash, just substance. You trust this person because the work speaks.
- **Theme:** Light/dark with existing teal-amber-green palette on slate. Both modes are well-established.
- **Typography:** Space Grotesk headings, Inter body, JetBrains Mono code. (Note: Inter and Space Grotesk are on the Impeccable reflex-reject list. These are established brand fonts, not new choices. Evaluate whether they serve the brand or should be reconsidered in a future typography pass.)

### Anti-References (ALL of these)
- **SaaS landing page:** No hero metrics, pricing grids, social proof walls, "trusted by" logos
- **Developer docs site:** Not bland or institutional. Personal and opinionated, not reference material.
- **AI startup splash:** No glowing gradients, dark mode neon, "the future is here" vibes
- **Course marketplace:** Not Udemy/Teachable. No lesson counts, star ratings, "enroll now" pressure.

### Design Principles
1. **Substance over spectacle.** Every element earns its place. If it doesn't help the user do something, remove it.
2. **Show the work.** Demonstrate capability through real artifacts (tools, exercises, content) not through claims about capability.
3. **Respect the practitioner.** No hand-holding, no marketing speak, no artificial urgency. Trust that the audience is smart.
4. **Warm precision.** Technical but never cold. Opinionated but never dismissive. The details matter because the people matter.
5. **One-of-one.** This site should not look like it came from a template, a course, or an AI. It should look like it was built by someone with taste and opinions.
